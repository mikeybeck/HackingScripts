c99 shell (encrypted)

24 Jan 2014

Many hack scripts are encoded before being uploaded and executed by the server under attack. PHP includes a native base 64 encryption function (base64_encode()), which is often used with a zip function (gzdeflate()) and a string rotation function (str_rot13()) to encode a shell script and make it difficult for people to read.
The script is then uploaded by a potential attacker, and the server decrypts and executes it. The most obvious telltale sign of an encrypted shell script is having ‘eval(gzinflate(base64_decode’ at the beginning, followed by a whole lot of what looks like random characters.
